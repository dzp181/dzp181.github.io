易GO项目总结

发表于 2017-05-25

昨天项目已经提交审核了，这个共享汽车项目就到此完成一个版本了。由此记录一点心得。。。

（一）、项目总结

项目开发背景
  1. 易GO出行app第一个版本。3月份开始接受项目，从项目评审到项目搭建，再到提测，最后到上线，这中间发生了太多事情。讲真共享汽车真不好做。而这段时间许多公司都出了共享汽车的APP，简直压力很大有没有！
  2. 易GO出行app第一个版本是仿照其他一个软件做的，这种大环境下，好像共享自行车都是仿照市场流行的app做的。（难道是行业不成文的套路？）
  3. 易GO出行项目硬件问题也是一个很棘手的问题。防盗，计费，归还…..都是需要考虑且要解决的事情。毕竟成本在那里。
项目开发技术
  1. 易GO出行项目开发主要运用的技术是地图相关的。地位，导航，线路规划都是要一整套的。
  2. 易GO出行项目开发需要一个转场动画。相关技术资料
  3. 易GO出行项目开发配合雷达的推送也是一个很复杂的技术点。设置时间搜索附近的停车场，根据业务去设置推送。
  4. runtime动态绑定属性还是很方便的。对地图的大头针绑定停车场ID和选择状态还真是一个很好的选择。
  5. runtime 方法交换和单点登录等。
